# US-008: Acceso al expediente y visualización de solicitud inicial desde Tramitagune

## ID
US-008

## Título
Acceso al expediente y visualización del formulario de datos

## Como
Funcionario tramitador

## Quiero
Acceder al expediente y visualizar el formulario con los datos cumplimentados por el ciudadano

## Para
Revisar la información del formulario y determinar si es necesario marcar campos para subsanación

## Descripción funcional

El funcionario tramitador accede a los expedientes pendientes de revisión. Selecciona un expediente específico y el sistema muestra únicamente el formulario de datos (paso 2) con todos los campos cumplimentados por el ciudadano. No se muestran otros pasos del proceso como identificación de participantes, documentos o declaraciones. El sistema presenta el formulario organizado por secciones para facilitar la revisión de los datos.

## Flujo principal

1. Tramitador accede al sistema de tramitación
2. Sistema muestra lista de expedientes pendientes de revisión
3. Tramitador selecciona expediente específico
4. Sistema muestra encabezado con datos del procedimiento: nombre, organismo, fecha de presentación
5. Sistema muestra formulario de datos (paso 2) organizado por secciones
6. Para cada campo muestra: etiqueta, valor introducido por el ciudadano, indicación si era obligatorio
7. Sistema NO permite cambiar a otros pasos ni visualizar documentos o declaraciones
8. Tramitador puede desplazarse por las secciones del formulario para revisar todos los campos

## Criterios de aceptación

### Escenario 1: Visualización del formulario de datos

**Dado** un funcionario tramitador autenticado  
**Cuando** selecciona un expediente pendiente de revisión  
**Entonces** el sistema muestra encabezado con datos del procedimiento  
**Y** muestra el formulario de datos (paso 2) organizado por secciones  
**Y** muestra todos los campos del formulario con sus valores  
**Y** NO muestra otros pasos como identificación, documentos o declaraciones

### Escenario 2: Navegación por secciones del formulario

**Dado** un formulario con múltiples secciones de campos  
**Cuando** el tramitador visualiza el formulario  
**Entonces** el sistema muestra claramente las diferentes secciones  
**Y** permite desplazarse entre secciones del formulario  
**Y** mantiene visible todos los campos de cada sección  
**Y** NO permite cambiar a otros pasos del proceso

### Escenario 3: Identificación de campos obligatorios

**Dado** un formulario con campos obligatorios y opcionales  
**Cuando** el tramitador visualiza el formulario  
**Entonces** el sistema distingue visualmente los campos obligatorios  
**Y** muestra el valor introducido para cada campo  
**Y** permite identificar fácilmente campos vacíos u omitidos

### Escenario 4: Selección de diferentes expedientes

**Dado** un tramitador revisando un expediente  
**Cuando** termina la revisión y quiere revisar otro  
**Entonces** el sistema permite volver a la lista de expedientes  
**Y** permite seleccionar otro expediente  
**Y** muestra el formulario del nuevo expediente seleccionado

### Escenario 5: Visualización de campos vacíos

**Dado** un formulario con algunos campos sin cumplimentar  
**Cuando** el tramitador visualiza el formulario  
**Entonces** el sistema muestra claramente los campos vacíos  
**Y** indica si un campo vacío era obligatorio  
**Y** permite al tramitador identificar fácilmente información faltante
