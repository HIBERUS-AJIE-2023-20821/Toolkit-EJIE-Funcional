# US-016: Visualización condicional del bloque "Otra documentación"

## ID
US-016

## Título
Visualización condicional del bloque "Otra documentación"

## Como
Ciudadano que tramita un procedimiento

## Quiero
Ver solo las opciones de aportación documental permitidas para mi trámite

## Para
No confundirme con opciones que no puedo utilizar y aportar únicamente los documentos permitidos por el procedimiento

## Descripción funcional

El sistema debe consultar la configuración del procedimiento en el CCP (Catálogo de Configuración de Procedimientos) para determinar si está permitido aportar "otros documentos" en el paso 4. En función de esta configuración, el bloque de "Otra documentación" se mostrará u ocultará dinámicamente al ciudadano.

Si el procedimiento tiene configurado que permite aportar "otros documentos", el bloque debe visualizarse en el paso 4, permitiendo al ciudadano seleccionar y cargar documentación adicional no contemplada en los documentos obligatorios.

Si la configuración indica que no se pueden aportar "otros documentos", el bloque no debe aparecer en la interfaz del paso 4, y el ciudadano solo podrá aportar los documentos explícitamente requeridos por el procedimiento.

Esta funcionalidad garantiza que la experiencia del ciudadano se adapte automáticamente a las reglas de cada procedimiento, evitando confusión y errores en la aportación documental.

## Flujo principal

1. Ciudadano accede al paso 4 - Documentos de la solicitud
2. Sistema consulta configuración del procedimiento en el catálogo
3. Sistema verifica si el procedimiento permite aportar "otros documentos"
4. Si está permitido, sistema muestra bloque "Otra documentación" en la interfaz
5. Si no está permitido, sistema oculta bloque "Otra documentación"
6. Ciudadano visualiza solo las opciones de aportación documental configuradas para el procedimiento

## Criterios de aceptación

### Escenario 1: Procedimiento permite otros documentos

**Dado** un procedimiento con configuración que permite aportar "otros documentos"  
**Cuando** el ciudadano accede al paso 4 - Documentos  
**Entonces** el bloque "Otra documentación" se muestra en la interfaz  
**Y** el ciudadano puede aportar documentación adicional

### Escenario 2: Procedimiento no permite otros documentos

**Dado** un procedimiento con configuración que no permite aportar "otros documentos"  
**Cuando** el ciudadano accede al paso 4 - Documentos  
**Entonces** el bloque "Otra documentación" no aparece en la interfaz  
**Y** el ciudadano solo ve los documentos obligatorios del procedimiento

### Escenario 3: Configuración consultada correctamente

**Dado** el ciudadano iniciando el paso 4  
**Cuando** el sistema carga la configuración del procedimiento  
**Entonces** el sistema consulta correctamente el catálogo de procedimientos  
**Y** obtiene la configuración sobre "otros documentos"  
**Y** aplica la visualización correspondiente antes de mostrar la interfaz

### Escenario 4: Cambio de configuración entre procedimientos

**Dado** un ciudadano que tramita diferentes procedimientos  
**Cuando** accede al paso 4 de procedimientos con configuraciones diferentes  
**Entonces** el bloque "Otra documentación" aparece solo en procedimientos que lo permiten  
**Y** la visualización se adapta dinámicamente a cada procedimiento

### Escenario 5: Visualización consistente durante toda la tramitación

**Dado** un procedimiento con configuración específica sobre "otros documentos"  
**Cuando** el ciudadano avanza y retrocede entre pasos de la solicitud  
**Entonces** la visualización del bloque "Otra documentación" se mantiene consistente  
**Y** no cambia durante el proceso de tramitación de esa solicitud
