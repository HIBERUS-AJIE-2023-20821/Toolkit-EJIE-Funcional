# US-019: Visualización de documentos cedentes

**Épica:** [EPIC-006](EPIC-006.md) - Autorizaciones de consulta de datos

## Historia de usuario
**Como** ciudadano  
**Quiero** ver qué organismo cede cada dato  
**Para** entender de dónde proviene la información que se va a consultar

## Descripción funcional
Para cada dato que la Administración puede consultar, debe mostrarse claramente el organismo o entidad cedente. Esta información permite al ciudadano comprender el origen de los datos y tomar decisiones informadas sobre autorizar o no cada consulta.

## Criterios de aceptación

### CA-1: Etiqueta de cedente visible
**Dado** que se muestra la lista de datos a consultar  
**Cuando** el ciudadano visualiza cada elemento  
**Entonces** debe incluir una etiqueta o indicador que muestre el organismo cedente

### CA-2: Formato de presentación
**Dado** que se muestra un dato con su cedente  
**Cuando** el ciudadano lo visualiza  
**Entonces** debe aparecer claramente:
- Nombre o descripción del dato
- Etiqueta "Cedente" u organismo que proporciona el dato

### CA-3: Todos los datos tienen cedente
**Dado** que se presenta la lista de autorizaciones  
**Cuando** el sistema muestra los datos a consultar  
**Entonces** cada dato debe indicar su organismo cedente

## Reglas de negocio
- **RN-1**: Todo dato a consultar debe tener asociado un organismo cedente
- **RN-2**: El nombre del cedente debe ser comprensible para el ciudadano
- **RN-3**: La información del cedente debe estar visible sin necesidad de interacción adicional
