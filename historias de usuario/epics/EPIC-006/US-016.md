# US-016: Presentación de autorizaciones configuradas

**Épica:** [EPIC-006](EPIC-006.md) - Autorizaciones de consulta de datos

## Historia de usuario
**Como** ciudadano que tramita una solicitud  
**Quiero** ver claramente qué datos puede consultar la Administración en mi nombre  
**Para** decidir si autorizo o me opongo a cada consulta

## Descripción funcional
En el paso de autorizaciones, el sistema debe presentar al ciudadano todas las autorizaciones configuradas para el procedimiento, organizadas por tipo:
- Autorizaciones por Ley (consulta obligatoria por mandato legal)
- Autorizaciones salvo oposición (consulta autorizada por defecto)
- Autorizaciones únicamente si consiente (requiere autorización expresa)

Para cada autorización debe mostrarse:
- Descripción clara del dato a consultar
- Organismo o entidad cedente
- Tipo de autorización según su naturaleza jurídica

## Criterios de aceptación

### CA-1: Visualización del paso de autorizaciones
**Dado** que el ciudadano ha completado los pasos previos  
**Cuando** accede al paso de autorizaciones  
**Entonces** debe visualizar todas las autorizaciones configuradas para el procedimiento

### CA-2: Información de consulta de datos generales
**Dado** que existen autorizaciones de tipo "salvo oposición"  
**Cuando** se muestra el paso de autorizaciones  
**Entonces** debe presentar:
- Título "Consulta de datos de la Administración"
- Explicación de que se consultarán datos para evitar aportar documentación
- Lista de datos que se pueden consultar
- Organismo cedente de cada dato

### CA-3: Información de consulta de datos tributarios
**Dado** que existen autorizaciones de tipo "únicamente si consiente" para datos tributarios  
**Cuando** se muestra el paso de autorizaciones  
**Entonces** debe presentar:
- Título "Consulta de datos tributarios"
- Explicación de que se requiere autorización expresa
- Lista de datos tributarios que se pueden consultar
- Organismo cedente de cada dato

### CA-4: Información de consultas por Ley
**Dado** que existen autorizaciones de tipo "por Ley"  
**Cuando** se muestra el paso de autorizaciones  
**Entonces** debe presentar:
- Título "Consultas establecidas por Ley"
- Explicación de que la consulta se realizará en cumplimiento de la normativa
- Lista de datos que se consultarán
- Organismo cedente de cada dato
- Indicación de que no es posible oponerse

### CA-5: Sin autorizaciones configuradas
**Dado** que el procedimiento no tiene autorizaciones configuradas  
**Cuando** el ciudadano accede al paso de autorizaciones  
**Entonces** el sistema debe omitir este paso y continuar con el siguiente

## Reglas de negocio
- **RN-1**: Las autorizaciones se configuran a nivel de procedimiento
- **RN-2**: Debe mostrarse el organismo cedente de cada dato
- **RN-3**: La información debe ser clara y comprensible para el ciudadano
- **RN-4**: Si no hay autorizaciones, el paso no se muestra
