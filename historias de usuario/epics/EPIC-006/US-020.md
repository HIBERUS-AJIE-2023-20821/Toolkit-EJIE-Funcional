# US-020: Inclusión del motivo de oposición en el detailInfo del ejgvDocument

**Épica:** [EPIC-006](EPIC-006.md) - Autorizaciones de consulta de datos

## Historia de usuario
**Como** sistema de presentación de solicitudes  
**Quiero** recibir el motivo de oposición del ciudadano en el campo `motivoOposicion` del `detailInfo`  
**Para** que el registro oficial de la solicitud contenga la justificación de la oposición a la consulta de datos

## Descripción funcional
Cuando el ciudadano se opone a la consulta de una o más autorizaciones del bloque "Datos salvo oposición" e introduce el motivo, esta información debe incluirse en el XML `detailInfo` que forma parte del `ejgvDocument` enviado al PPS en el momento de la presentación.

El campo que trasladará esta información es `motivoOposicion`, que:
- Se incluye cuando existe motivo de oposición registrado
- Se omite cuando no existe oposición ni motivo

## Criterios de aceptación

### CA-1: Campo motivoOposicion incluido cuando existe oposición
**Dado** que el ciudadano ha marcado oposición a al menos una autorización y ha introducido el motivo  
**Cuando** el sistema envía el `ejgvDocument` al PPS  
**Entonces** el `detailInfo` del XML incluye el campo `motivoOposicion` con el texto introducido por el ciudadano

### CA-2: Campo motivoOposicion omitido cuando no hay oposición
**Dado** que el ciudadano no ha marcado oposición a ninguna autorización  
**Cuando** el sistema envía el `ejgvDocument` al PPS  
**Entonces** el `detailInfo` del XML no incluye el campo `motivoOposicion`

### CA-3: Coherencia con los datos registrados
**Dado** que el ciudadano ha introducido un motivo de oposición  
**Cuando** se genera el `detailInfo`  
**Entonces** el valor del campo `motivoOposicion` coincide exactamente con el texto introducido por el ciudadano en el formulario
