# US-020: Validación de autorizaciones antes de continuar

**Épica:** [EPIC-006](EPIC-006.md) - Autorizaciones de consulta de datos

## Historia de usuario
**Como** sistema  
**Quiero** validar que el ciudadano ha completado todas las autorizaciones obligatorias  
**Para** garantizar que puede continuar con el proceso de tramitación

## Descripción funcional
Antes de permitir avanzar al siguiente paso (Documentación), el sistema debe verificar que el ciudadano ha gestionado todas las autorizaciones obligatorias. En particular:
- Para autorizaciones tipo "únicamente si consiente", debe haber seleccionado una opción (sí o no)
- Para autorizaciones tipo "salvo oposición", puede continuar (el valor por defecto es autorizar)

## Criterios de aceptación

### CA-1: Validación de autorizaciones obligatorias
**Dado** que existen autorizaciones de tipo "únicamente si consiente"  
**Cuando** el ciudadano intenta continuar sin seleccionar una opción  
**Entonces** el sistema debe mostrar un mensaje de error indicando que debe tomar una decisión

### CA-2: Identificación de campos pendientes
**Dado** que hay autorizaciones obligatorias sin completar  
**Cuando** el sistema muestra el mensaje de validación  
**Entonces** debe indicar claramente qué autorizaciones faltan por responder

### CA-3: Permitir continuar con oposición
**Dado** que el ciudadano se opuso a consultas tipo "salvo oposición"  
**Cuando** intenta continuar al siguiente paso  
**Entonces** el sistema debe permitirlo (la oposición es válida)

### CA-4: Permitir continuar sin autorizar tributarios
**Dado** que el ciudadano seleccionó "No autorizo" en datos tributarios  
**Cuando** intenta continuar  
**Entonces** el sistema debe permitirlo (ha tomado una decisión)

### CA-5: Registro de decisiones
**Dado** que el ciudadano completa el paso de autorizaciones  
**Cuando** avanza al siguiente paso  
**Entonces** el sistema debe registrar todas las decisiones tomadas

### CA-6: Impacto en paso de documentos
**Dado** que el ciudadano se opuso o no autorizó consultas  
**Cuando** accede al paso de documentos  
**Entonces** los documentos relacionados deben aparecer como obligatorios

## Reglas de negocio
- **RN-1**: Las autorizaciones "únicamente si consiente" son obligatorias de responder
- **RN-2**: Las autorizaciones "salvo oposición" no bloquean (por defecto se autoriza)
- **RN-3**: Las decisiones de autorización afectan a los documentos obligatorios
- **RN-4**: Todas las decisiones se registran para trazabilidad
