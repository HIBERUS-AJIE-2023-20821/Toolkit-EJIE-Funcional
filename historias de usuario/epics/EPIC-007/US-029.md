# US-029: Descarga y presentación de solicitud presencial

**Épica:** [EPIC-007](EPIC-007.md) - Presencial Ciudadanía - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano que ha generado una solicitud presencial  
**Quiero** descargar el PDF completo o solo el código QR  
**Para** presentarlo posteriormente en las oficinas de la Administración

## Descripción funcional
En la pantalla final de éxito, tras generar la solicitud presencial y el PDF con QR integrado, el sistema ofrece al ciudadano dos opciones de descarga:

1. **Descargar solicitud completa**: Descarga el PDF completo con todos los datos y el código QR integrado
2. **Descargar solo QR**: Descarga únicamente la imagen o documento del código QR

La primera opción permite al ciudadano tener un documento completo con toda la información que ha introducido. La segunda opción facilita casos donde solo necesita el código para recuperación rápida, sin imprimir todo el documento.

El ciudadano puede elegir entre presentarse con:
- El PDF completo impreso + documentación física adicional si tiene
- Solo el QR impreso o en dispositivo móvil + documentación física
- Ambos documentos según su preferencia

La pantalla de finalización muestra instrucciones claras con los siguientes pasos:
1. Acudir dentro del plazo a uno de los lugares de presentación establecidos (debe acudir la persona que figura en la solicitud)
2. Mostrar DNI/NIE para que el tramitador pueda acceder a la solicitud
3. Firmar la solicitud presencialmente
4. Entregar la documentación no aportada en el paso de documentos del formulario

La pantalla muestra el código QR visualmente y ofrece opciones para descargar tanto la solicitud completa como el código QR por separado.

## Flujo principal
1. Sistema muestra pantalla de finalización tras generar solicitud presencial
2. Sistema presenta mensaje de atención con 4 pasos para presentación presencial
3. Sistema muestra código QR visualmente en pantalla
4. Sistema ofrece botón "Descargar la solicitud" y botón "Descargar código QR"
5. Ciudadano elige descargar solicitud completa o solo QR
6. Sistema inicia descarga del archivo seleccionado
7. Sistema guarda archivo en dispositivo del ciudadano
8. Ciudadano puede descargar ambas opciones si lo desea
9. Ciudadano cierra la sesión

## Criterios de aceptación

### Escenario 1: Visualización de opciones de descarga

**Dado** que la solicitud presencial se ha generado exitosamente  
**Cuando** el ciudadano está en la pantalla de finalización  
**Entonces** el sistema muestra opción "Descargar solicitud"  
**Y** el sistema muestra opción "Descargar QR"  
**Y** ambas opciones están claramente diferenciadas  
**Y** el ciudadano puede elegir una o ambas

### Escenario 2: Descarga del PDF completo

**Dado** que el ciudadano selecciona "Descargar solicitud"  
**Cuando** confirma la acción  
**Entonces** el sistema descarga el PDF completo  
**Y** el PDF incluye todos los datos y el QR integrado  
**Y** el nombre del archivo es descriptivo e incluye código de solicitud presencial  
**Y** el formato es PDF estándar

### Escenario 3: Descarga solo del código QR

**Dado** que el ciudadano selecciona "Descargar QR"  
**Cuando** confirma la acción  
**Entonces** el sistema descarga imagen con el código QR  
**Y** el formato del archivo es PNG o JPEG  
**Y** la imagen incluye el QR y el código en formato texto  
**Y** el tamaño es apropiado para escaneo

### Escenario 4: Descarga de ambas opciones

**Dado** que el ciudadano está en pantalla de éxito  
**Cuando** descarga primero una opción y luego la otra  
**Entonces** el sistema permite descargar ambas  
**Y** cada descarga funciona independientemente  
**Y** no hay límite en número de descargas

### Escenario 5: Instrucciones de siguientes pasos

**Dado** que el ciudadano está en pantalla de finalización  
**Cuando** visualiza la pantalla  
**Entonces** el sistema muestra mensaje "Para que esta solicitud quede registrada en la Administración, deberá usted seguir estos pasos"  
**Y** presenta los 4 pasos numerados: acudir a lugar de presentación, mostrar DNI/NIE, firmar solicitud, entregar documentación  
**Y** destaca que debe acudir la persona que figura en la solicitud  
**Y** indica qué documentación debe entregar físicamente

### Escenario 6: Información del código de solicitud presencial

**Dado** que el ciudadano está en pantalla de finalización  
**Cuando** visualiza la información  
**Entonces** el sistema muestra el código de solicitud presencial en texto claro  
**Y** el ciudadano puede copiar o anotar el código manualmente  
**Y** se informa de la importancia de conservar el código

### Escenario 7: Visualización del código QR en pantalla

**Dado** que la solicitud presencial se ha generado exitosamente  
**Cuando** el ciudadano visualiza la pantalla de finalización  
**Entonces** el sistema muestra el código QR de forma visual  
**Y** incluye texto explicativo sobre el uso del QR  
**Y** indica dónde presentarlo  
**Y** el QR es visible sin necesidad de descarga

### Escenario 8: Cierre de sesión tras descarga

**Dado** que el ciudadano ha descargado los documentos  
**Cuando** cierra el navegador o la sesión  
**Entonces** el sistema finaliza la sesión de forma segura  
**Y** los datos temporales se limpian  
**Y** no queda información sensible en el navegador
