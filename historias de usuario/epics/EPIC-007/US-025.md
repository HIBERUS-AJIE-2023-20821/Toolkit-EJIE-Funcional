# US-025: Acceso sin autenticación para presencial ciudadanía

**Épica:** [EPIC-007](EPIC-007.md) - Presencial Ciudadanía - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano que desea iniciar una solicitud presencial  
**Quiero** acceder al formulario sin necesidad de autenticarme  
**Para** comenzar a rellenar mi solicitud sin requerir certificado digital o identificación electrónica

## Descripción funcional
El ciudadano accede al formulario de tramitación mediante URL que incluye el código del procedimiento y el identificador de acción telemática con valor 19 (tipoPresentación=19), que identifica el modo presencial ciudadanía.

El sistema recibe estos parámetros y reconoce que se trata de una solicitud presencial, omitiendo completamente el proceso de autenticación electrónica que sería obligatorio en una tramitación telemática estándar.

Una vez identificado el modo presencial, el sistema presenta directamente la **pantalla de selección de tipo de tramitación** con dos opciones:
- **Para mí**: El ciudadano tramita para sí mismo
- **Para otros**: El ciudadano tramita en representación de otra persona

Esta pantalla es el punto de entrada tras el acceso sin autenticación, permitiendo al ciudadano indicar el tipo de tramitación antes de comenzar con los pasos del formulario.

**Importante**: La identificación de los interesados del expediente se realizará mediante **formulario de introducción manual de datos**. El sistema **no consultará** el Registro de Interesados (RI) ni el Registro de Apoderamientos (RdA) en ningún momento durante el proceso.

## Flujo principal
1. Ciudadano accede a URL con código de procedimiento e idAcciónTelemática=19
2. Sistema recibe parámetros de acceso
3. Sistema identifica modo presencial ciudadanía (tipoPresentación=19)
4. Sistema omite proceso de autenticación electrónica
5. Sistema presenta pantalla "¿Cómo desea proceder con la tramitación?"
6. Sistema muestra opciones: "Para mí" / "Para otros"
7. Ciudadano selecciona tipo de tramitación

## Criterios de aceptación

### CA-1: Acceso con tipo presentación presencial
**Dado** que el ciudadano accede mediante URL con procedimiento e idAcciónTelemática=19  
**Cuando** el sistema procesa los parámetros de acceso  
**Entonces** el sistema identifica modo presencial ciudadanía  
**Y** omite el proceso de autenticación electrónica

### CA-2: Omisión de autenticación
**Dado** que el sistema ha identificado tipoPresentación=19  
**Cuando** carga la aplicación  
**Entonces** no solicita certificado digital  
**Y** no solicita identificación electrónica  
**Y** no muestra pantalla de autenticación

### CA-3: Presentación de pantalla de selección
**Dado** que el acceso es sin autenticación  
**Cuando** el sistema completa la carga inicial  
**Entonces** presenta la pantalla "¿Cómo desea proceder con la tramitación?"  
**Y** muestra la opción "Para mí"  
**Y** muestra la opción "Para otros"

### CA-4: Validación de parámetros obligatorios
**Dado** que el ciudadano intenta acceder  
**Cuando** la URL no incluye idAcciónTelemática=19  
**Entonces** el sistema requiere autenticación estándar  
**Y** no permite acceso sin autenticación

### CA-5: Registro del modo presencial
**Dado** que el sistema identifica tipoPresentación=19  
**Cuando** inicia la sesión  
**Entonces** registra el modo presencial ciudadanía en la sesión  
**Y** mantiene esta configuración durante toda la tramitación
