# US-025: Acceso sin autenticación para presencial ciudadanía

**Épica:** [EPIC-007](EPIC-007.md) - Presencial Ciudadanía - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano que desea iniciar una solicitud presencial  
**Quiero** acceder al formulario sin necesidad de autenticarme  
**Para** comenzar a rellenar mi solicitud sin requerir certificado digital o identificación electrónica

## Descripción funcional
El ciudadano accede al formulario de tramitación con un parámetro específico en la URL (tipoPresentación=19) que indica que se trata de una solicitud presencial ciudadanía. Este modo permite iniciar el formulario sin requerir autenticación previa mediante certificado digital o identificación electrónica.

El sistema identifica que se trata de un acceso presencial y omite los pasos de autenticación que normalmente serían obligatorios en una tramitación telemática estándar. El ciudadano puede comenzar a rellenar el formulario directamente sin barreras de acceso.

Este acceso sin autenticación está limitado exclusivamente a solicitudes presenciales para personas físicas, donde el ciudadano tramita para sí mismo o como representante sin registro de apoderamientos. No está permitido para tramitaciones de empresas ni para representaciones formales registradas.

## Flujo principal
1. Ciudadano accede a URL de procedimiento con parámetro tipoPresentación=19
2. Sistema identifica modo presencial ciudadanía
3. Sistema omite pantalla de autenticación
4. Sistema permite acceso directo al formulario
5. Ciudadano visualiza paso 1 del formulario sin estar autenticado

## Criterios de aceptación

### Escenario 1: Acceso con tipo presentación presencial

**Dado** que el ciudadano accede con tipoPresentación=19  
**Cuando** el sistema carga la URL del procedimiento  
**Entonces** el sistema omite la pantalla de autenticación  
**Y** permite el acceso directo al formulario

### Escenario 2: Identificación del modo presencial

**Dado** que el sistema recibe tipoPresentación=19  
**Cuando** se inicia la sesión  
**Entonces** el sistema registra que es modo presencial ciudadanía  
**Y** adapta el flujo de tramitación en consecuencia

### Escenario 3: Limitación a personas físicas

**Dado** que el acceso es modo presencial ciudadanía  
**Cuando** el ciudadano inicia la solicitud  
**Entonces** el sistema solo permite tramitación para personas físicas  
**Y** no permite tramitación para empresas  
**Y** no permite representación con registro de apoderamientos

### Escenario 4: Validación del parámetro tipoPresentación

**Dado** que el ciudadano accede sin tipoPresentación=19  
**Cuando** intenta acceder al formulario  
**Entonces** el sistema requiere autenticación estándar  
**Y** no permite acceso sin autenticación
