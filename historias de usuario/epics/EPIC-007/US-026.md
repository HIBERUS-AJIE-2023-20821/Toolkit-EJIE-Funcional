# US-026: Rellenado de formulario y carga de documentos

**Épica:** [EPIC-007](EPIC-007.md) - Presencial Ciudadanía - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano que inicia una solicitud presencial  
**Quiero** rellenar el formulario y cargar documentos desde el equipo público  
**Para** aportar toda la información disponible antes de presentarme ante el tramitador

## Descripción funcional
Una vez accedido al formulario sin autenticación, el ciudadano recorre los pasos de tramitación completando los datos necesarios y cargando documentos desde el dispositivo local.

El flujo de rellenado es similar al de una tramitación telemática estándar, pero adaptado al contexto presencial:
- No se solicita firma electrónica en ningún paso
- No hay validaciones que requieran datos de certificados digitales
- Se permite guardar la información en cualquier momento antes de finalizar
- Los documentos se almacenan en Dokusi y quedan relacionados con la solicitud
- Las limitaciones de documentos (número y tamaño) son las mismas que en tramitación estándar

El ciudadano puede completar todos los pasos o solo algunos, guardando la información disponible. El objetivo es facilitar que el ciudadano aporte el máximo de datos posible antes de la presentación presencial, reduciendo el trabajo posterior del tramitador.

## Flujo principal
1. Ciudadano accede a los pasos del formulario en modo presencial
2. Sistema muestra pasos adaptados a modo presencial
3. Ciudadano rellena campos de datos
4. Ciudadano carga documentos desde dispositivo local si dispone de ellos
5. Sistema valida formatos y tamaños de documentos
6. Sistema almacena documentos en Dokusi relacionados con la solicitud
7. Ciudadano revisa autorizaciones y declaraciones si aplican
8. Sistema guarda información temporalmente en cada paso

## Criterios de aceptación

### Escenario 1: Rellenado de datos personales sin autenticación

**Dado** que el ciudadano está en modo presencial  
**Cuando** accede al paso de datos del solicitante  
**Entonces** el sistema permite introducir datos manualmente  
**Y** no prerellena datos de certificado digital  
**Y** todos los campos son editables

### Escenario 2: Carga de documentos y almacenamiento en Dokusi

**Dado** que el ciudadano está en el paso de documentos  
**Cuando** selecciona archivos desde el dispositivo  
**Entonces** el sistema permite cargar documentos  
**Y** valida formato y tamaño según limitaciones estándar  
**Y** almacena los documentos en Dokusi  
**Y** relaciona los documentos con la solicitud presencial

### Escenario 3: Navegación entre pasos sin firma

**Dado** que el ciudadano rellena el formulario presencial  
**Cuando** navega entre pasos  
**Entonces** el sistema no solicita firma en ningún momento  
**Y** permite avanzar y retroceder libremente

### Escenario 4: Validaciones de campos obligatorios

**Dado** que el ciudadano rellena campos  
**Cuando** intenta avanzar de paso  
**Entonces** el sistema valida campos obligatorios del paso actual  
**Y** muestra mensajes de validación si faltan datos  
**Y** permite continuar si los datos son válidos

### Escenario 5: Guardado de progreso

**Dado** que el ciudadano rellena información  
**Cuando** completa cada campo o paso  
**Entonces** el sistema guarda temporalmente la información  
**Y** mantiene los datos en la sesión activa

### Escenario 6: Limitación de autorizaciones y declaraciones

**Dado** que el ciudadano está en pasos de autorizaciones o declaraciones  
**Cuando** visualiza las opciones  
**Entonces** el sistema muestra solo lo aplicable a modo presencial  
**Y** no solicita firma electrónica de declaraciones  
**Y** registra las decisiones del ciudadano
