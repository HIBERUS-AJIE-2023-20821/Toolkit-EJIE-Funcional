# US-026: Rellenado de formulario y carga de documentos

**Épica:** [EPIC-007](EPIC-007.md) - Presencial Ciudadanía - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano que inicia una solicitud presencial  
**Quiero** rellenar el formulario y cargar documentos desde el equipo público  
**Para** aportar toda la información disponible antes de presentarme ante el tramitador

## Descripción funcional
Una vez seleccionado el tipo de tramitación ("Para mí" o "Para otros"), el ciudadano accede a los pasos del formulario donde debe introducir manualmente todos los datos requeridos.

**Identificación de interesados**: La identificación de los interesados del expediente (solicitante y, en su caso, representados) se realiza mediante **formulario de introducción manual de datos**. El sistema **no consulta** en ningún momento:
- **RI (Registro de Interesados)**: No se recuperan datos de interesados previamente registrados
- **RdA (Registro de Apoderamientos)**: No se valida ni recupera información de representaciones formales

Todos los datos de identificación deben ser introducidos manualmente por el ciudadano en cada solicitud presencial.

El flujo de rellenado es similar al de una tramitación telemática estándar, pero adaptado al contexto presencial:
- Introducción manual de todos los datos de identificación (solicitante e interesados)
- No se prerrellenan datos desde sistemas de registro (RI, RdA)
- No hay validaciones que requieran datos de certificados digitales
- **No procede guardado de borradores**: No se pueden guardar borradores intermedios
- **No procede uso de plantillas**: No se pueden cargar plantillas de solicitudes previas
- **Interfaz simplificada**: No se muestran enlaces a "Guardar borrador", "Cargar plantilla" ni funcionalidades relacionadas
- Se permite guardar la información temporalmente durante la sesión activa
- Los documentos se almacenan en Dokusi y quedan relacionados con la solicitud
- Las limitaciones de documentos (número y tamaño) son las mismas que en tramitación estándar

**El ciudadano debe completar todos los pasos obligatorios de la solicitud** antes de poder finalizarla. El objetivo es que el ciudadano aporte toda la información y documentación requerida antes de la presentación presencial, facilitando el trabajo posterior del tramitador.

## Flujo principal
1. Ciudadano selecciona tipo de tramitación ("Para mí" o "Para otros")
2. Sistema presenta pasos del formulario adaptados a modo presencial
3. Ciudadano introduce manualmente datos de identificación de interesados (solicitante y representados si aplica)
4. Sistema no consulta RI ni RdA durante la introducción de datos
5. Ciudadano rellena manualmente campos de datos específicos del procedimiento
6. Ciudadano carga documentos desde dispositivo local si dispone de ellos
7. Sistema valida formatos y tamaños de documentos
8. Sistema almacena documentos en Dokusi relacionados con la solicitud
9. Ciudadano revisa autorizaciones y declaraciones si aplican
10. Sistema guarda información temporalmente en cada paso

## Criterios de aceptación

### CA-1: Introducción manual de datos de interesados
**Dado** que el ciudadano ha seleccionado tipo de tramitación  
**Cuando** accede al paso de identificación de interesados  
**Entonces** el sistema presenta formulario de introducción manual de datos  
**Y** todos los campos están vacíos inicialmente  
**Y** el ciudadano debe completar manualmente todos los datos requeridos

### CA-2: No consulta a Registro de Interesados
**Dado** que el ciudadano está en modo presencial  
**Cuando** introduce datos de identificación  
**Entonces** el sistema no consulta el Registro de Interesados (RI)  
**Y** no prerellena datos desde registros previos  
**Y** no valida contra información del RI

### CA-3: No consulta a Registro de Apoderamientos
**Dado** que el ciudadano seleccionó "Para otros"  
**Cuando** introduce datos del representado  
**Entonces** el sistema no consulta el Registro de Apoderamientos (RdA)  
**Y** no valida la representación formal  
**Y** permite introducir cualquier representado manualmente

### CA-4: Campos editables sin prerrellenado
**Dado** que el ciudadano está en modo presencial  
**Cuando** accede a cualquier paso de datos  
**Entonces** todos los campos son editables  
**Y** no hay prerrellenado desde certificado digital  
**Y** no hay prerrellenado desde sistemas de registro

### CA-5: Carga de documentos y almacenamiento en Dokusi
**Dado** que el ciudadano está en el paso de documentos  
**Cuando** selecciona archivos desde el dispositivo  
**Entonces** el sistema permite cargar documentos  
**Y** valida formato y tamaño según limitaciones estándar  
**Y** almacena los documentos en Dokusi  
**Y** relaciona los documentos con la solicitud presencial

### CA-6: Navegación entre pasos
**Dado** que el ciudadano rellena el formulario presencial  
**Cuando** navega entre pasos  
**Entonces** el sistema permite avanzar y retroceder libremente entre los pasos disponibles

### CA-7: Validaciones de campos obligatorios

**Dado** que el ciudadano rellena campos  
**Cuando** intenta avanzar de paso  
**Entonces** el sistema valida campos obligatorios del paso actual  
**Y** muestra mensajes de validación si faltan datos  
**Y** permite continuar si los datos son válidos

### CA-8: Guardado de progreso temporal
**Dado** que el ciudadano rellena información  
**Cuando** completa cada campo o paso  
**Entonces** el sistema guarda temporalmente la información en la sesión activa  
**Y** mantiene los datos mientras la sesión permanece abierta  
**Y** los datos se pierden al cerrar la sesión

### CA-9: No visualización de funcionalidad de borradores
**Dado** que el ciudadano está en modo presencial  
**Cuando** visualiza cualquier pantalla del formulario  
**Entonces** el sistema no muestra enlace "Guardar borrador"  
**Y** no muestra enlace "Mis borradores"  
**Y** no permite guardar la solicitud como borrador recuperable

### CA-10: No visualización de funcionalidad de plantillas
**Dado** que el ciudadano está en modo presencial  
**Cuando** visualiza cualquier pantalla del formulario  
**Entonces** el sistema no muestra enlace "Cargar plantilla"  
**Y** no muestra enlace "Guardar como plantilla"  
**Y** no permite utilizar plantillas de solicitudes previas

### CA-11: Autorizaciones y declaraciones
**Dado** que el ciudadano está en pasos de autorizaciones o declaraciones  
**Cuando** visualiza las opciones  
**Entonces** el sistema muestra solo lo aplicable a modo presencial  
**Y** registra las decisiones del ciudadano

### CA-12: Completar todos los pasos obligatorios
**Dado** que el ciudadano está en modo presencial  
**Cuando** intenta finalizar la solicitud  
**Entonces** el sistema valida que todos los pasos obligatorios estén completos  
**Y** impide finalizar si algún paso obligatorio está incompleto  
**Y** muestra mensaje indicando qué pasos o datos faltan por completar
