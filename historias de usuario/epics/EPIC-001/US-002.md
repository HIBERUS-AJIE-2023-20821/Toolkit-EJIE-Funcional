# US-002: Visualización de datos del expediente y participantes

## ID
US-002

## Título
Visualización de datos del expediente y participantes

## Como
Ciudadano o representante

## Quiero
Visualizar los datos del expediente y de los participantes en modo solo lectura

## Para
Conocer el contexto de la subsanación y confirmar que estoy actuando en el expediente correcto

## Descripción funcional

El sistema debe mostrar de manera clara los datos del expediente (número, procedimiento, organismo, fecha de solicitud inicial, motivo de subsanación) y los datos de los participantes (interesado principal, representante si aplica). Debe identificar el tipo de representación (apoderamiento RdA o representación específica) y presentar toda la información en modo solo lectura. Los datos de contacto (email, teléfono) deben mostrarse como enlaces interactivos y debe diferenciarse visualmente entre interesado principal y representante actuante.

## Flujo principal

1. Sistema recupera datos del expediente desde sistemas de tramitación
2. Sistema recupera datos del interesado principal
3. Si existe representante, sistema obtiene datos y tipo de representación (RdA o específica)
4. Sistema muestra sección "Datos del expediente" con información contextual
5. Sistema muestra sección "Motivo de subsanación" con descripción del requerimiento
6. Sistema muestra sección "Participantes" diferenciando interesado y representante
7. Todos los datos se presentan en modo solo lectura con formato apropiado

## Criterios de aceptación

### Escenario 1: Visualización completa con representante

**Dado** un expediente con interesado y representante (RdA o específico)  
**Cuando** se muestra la información  
**Entonces** el sistema muestra:  
**Y** "Actuando como:" con indicación del rol y tipo de representación  
**Y** Sección "Interesado": nombre completo, NIF, email y teléfono  
**Y** Sección "Representante actuante": nombre completo, NIF, email y teléfono  
**Y** Indicación del tipo de representación (apoderamiento RdA o representación específica establecida en solicitud inicial)  
**Y** Todos los datos en modo solo lectura

### Escenario 2: Visualización como interesado principal

**Dado** un expediente donde el ciudadano es el interesado principal  
**Cuando** se muestra la información  
**Entonces** el sistema muestra "Actuando como: Interesado principal"  
**Y** muestra datos completos del interesado (nombre, NIF, email, teléfono)  
**Y** NO muestra sección de representante

### Escenario 3: Datos de contacto interactivos

**Dado** un participante con email y teléfono  
**Cuando** se visualiza la información  
**Entonces** el email se muestra como enlace clickable formato "mailto:"  
**Y** el teléfono se muestra como enlace clickable formato "tel:+34..."  
**Y** los enlaces funcionan correctamente en desktop y móvil

### Escenario 4: Información contextual del expediente

**Dado** un expediente con datos completos  
**Cuando** se muestra la sección de datos del expediente  
**Entonces** el sistema presenta:  
**Y** Número de expediente, procedimiento, código de procedimiento y organismo  
**Y** Fecha de solicitud inicial  
**Y** Motivo de subsanación destacado visualmente

### Escenario 5: Error al recuperar datos y diseño responsive

**Dado** un error al consultar sistemas de tramitación  
**Cuando** el sistema intenta recuperar datos de participantes  
**Entonces** reintenta hasta 3 veces  
**Y** si falla, muestra error "No se pudieron cargar los datos. Inténtalo de nuevo"  
**Y** proporciona botón "Reintentar"  
**Y** NO permite continuar hasta cargar datos correctamente  
**Y** en dispositivos móviles, la información se adapta con secciones apiladas verticalmente
