# US-012: Especificación de tipología y título del documento

## ID
US-012

## Título
Especificación de tipología y título del documento

## Como
Ciudadano solicitante

## Quiero
Especificar el tipo y un título descriptivo para cada documento que aporto

## Para
Identificar claramente cada documento adicional que incluyo en mi solicitud

## Descripción funcional

Antes de completar la carga de cada documento en "Otros documentos", el sistema solicita al ciudadano que especifique dos datos obligatorios: la tipología del documento (seleccionándola de una lista de tipos configurados para el procedimiento) y un título descriptivo en texto libre.

El selector de tipología presenta todos los tipos de documentos configurados para el procedimiento (no solo los de la acción telemática en curso), sin repeticiones. El campo de título permite al ciudadano introducir texto libre para identificar el contenido del documento. 

Si existen metadatos obligatorios asociados al tipo de documento seleccionado, el sistema también debe solicitarlos antes de completar la carga. Solo cuando toda la información obligatoria está cumplimentada, el documento se almacena con toda la información introducida y queda relacionado con la solicitud.

## Flujo principal

1. Ciudadano selecciona documento desde su PC
2. Sistema muestra formulario con campos obligatorios
3. Sistema presenta selector con todos los tipos de documentos configurados para el procedimiento
4. Ciudadano selecciona tipología de documento del listado
5. Sistema habilita campo de texto para título del documento
6. Ciudadano introduce título descriptivo en texto libre
7. Si la tipología seleccionada tiene metadatos obligatorios, sistema los solicita
8. Ciudadano completa metadatos obligatorios si los hay
9. Ciudadano confirma la información introducida
10. Sistema almacena el documento con toda la información asociada y lo relaciona con la solicitud

## Criterios de aceptación

### Escenario 1: Selección de tipología de documento

**Dado** un documento seleccionado para carga  
**Cuando** el sistema solicita información del documento  
**Entonces** presenta un selector con todas las tipologías configuradas para el procedimiento  
**Y** las tipologías aparecen sin repeticiones  
**Y** la selección de tipología es obligatoria

### Escenario 2: Introducción de título del documento

**Dado** la tipología de documento seleccionada  
**Cuando** el ciudadano accede al campo título  
**Entonces** puede introducir texto libre descriptivo  
**Y** el campo título es obligatorio  
**Y** debe completarse para continuar con la carga

### Escenario 3: Completar información obligatoria antes de carga

**Dado** un documento seleccionado desde PC  
**Cuando** el ciudadano intenta cargar el documento  
**Entonces** el sistema valida que tipología y título estén cumplimentados  
**Y** no permite completar la carga si falta información obligatoria  
**Y** muestra mensajes indicando los campos obligatorios pendientes

### Escenario 4: Metadatos obligatorios según tipología

**Dado** una tipología de documento con metadatos obligatorios configurados  
**Cuando** el ciudadano selecciona esa tipología  
**Entonces** el sistema muestra los campos de metadatos obligatorios  
**Y** requiere su cumplimentación antes de completar la carga  
**Y** no permite avanzar sin completar todos los metadatos obligatorios

### Escenario 5: Carga exitosa con información completa

**Dado** tipología seleccionada, título introducido y metadatos cumplimentados  
**Cuando** el ciudadano confirma la información  
**Entonces** el sistema almacena el documento con toda la información introducida  
**Y** el documento queda relacionado con la solicitud  
**Y** aparece listado en la sección "Otros documentos" del paso 4
