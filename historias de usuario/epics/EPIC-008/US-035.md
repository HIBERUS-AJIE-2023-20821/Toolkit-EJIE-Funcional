# US-035: Validación, firma y envío de subsanación

**Épica:** [EPIC-008](EPIC-008.md) - Subsanación - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano que ha completado las correcciones requeridas  
**Quiero** firmar y enviar la subsanación con validación de completitud  
**Para** finalizar el proceso y que mi expediente continúe su tramitación

## Descripción funcional
Una vez el ciudadano ha completado todas las correcciones requeridas, accede al paso final de firma y envío. 

Las validaciones de datos ya se han realizado en cada uno de los pasos anteriores según la configuración de obligatoriedad de cada elemento. El sistema no permite avanzar desde un paso si no se cumplen las validaciones requeridas. Por tanto, al llegar a este paso final, los datos ya están validados.

El sistema presenta la pantalla de firma. El ciudadano debe firmar electrónicamente la subsanación. Tras la firma, el sistema envía la subsanación, registra la acción y genera el justificante correspondiente que el ciudadano puede descargar.

El proceso de firma y envío sigue los mismos mecanismos que una solicitud normal, con integración con los servicios de firma electrónica y registro oficial.

## Flujo principal
1. Ciudadano completa todos los pasos de subsanación
2. Ciudadano accede al paso final de firma y envío
3. Sistema presenta pantalla de firma electrónica
4. Ciudadano firma electrónicamente la subsanación
5. Sistema envía la subsanación
6. Sistema registra la acción telemática
7. Sistema genera justificante de subsanación
8. Sistema envía justificante por correo electrónico
9. Ciudadano descarga justificante

## Criterios de aceptación

### Escenario 1: Presentación de pantalla de firma

**Dado** que el ciudadano accede al paso final  
**Cuando** ha completado todos los pasos anteriores  
**Entonces** el sistema presenta pantalla de firma electrónica  
**Y** muestra resumen de lo que se va a firmar  
**Y** ofrece opciones de firma disponibles

### Escenario 2: Firma electrónica de subsanación

**Dado** que el ciudadano accede a firma  
**Cuando** completa el proceso de firma  
**Entonces** el sistema aplica firma electrónica a la subsanación  
**Y** valida la firma correctamente  
**Y** registra datos del firmante

### Escenario 3: Envío tras firma exitosa

**Dado** que la subsanación está firmada  
**Cuando** el sistema procesa el envío  
**Entonces** el sistema registra la acción telemática  
**Y** actualiza el estado del expediente  
**Y** notifica al sistema tramitador

### Escenario 4: Generación de justificante

**Dado** que la subsanación se ha enviado correctamente  
**Cuando** el sistema completa el registro  
**Entonces** el sistema genera justificante de subsanación  
**Y** incluye código de registro  
**Y** incluye resumen de elementos subsanados  
**Y** permite descarga inmediata  
**Y** envía el justificante por correo electrónico al ciudadano

### Escenario 5: Gestión de errores en envío

**Dado** que ocurre un error durante el envío  
**Cuando** el sistema no puede completar el registro  
**Entonces** el sistema muestra mensaje de error  
**Y** mantiene la subsanación firmada  
**Y** permite reintentar el envío  
**Y** no pierde los cambios realizados

### Escenario 6: Confirmación de subsanación completada

**Dado** que la subsanación se envía exitosamente  
**Cuando** el sistema confirma el proceso  
**Entonces** el sistema muestra pantalla de confirmación  
**Y** indica que el expediente continuará su tramitación  
**Y** proporciona código de referencia  
**Y** ofrece descarga del justificante
