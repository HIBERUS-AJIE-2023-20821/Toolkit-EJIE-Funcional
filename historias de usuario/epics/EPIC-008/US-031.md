# US-031: Carga de paso 2 con datos de solicitud original

**Épica:** [EPIC-008](EPIC-008.md) - Subsanación - Proceso del Ciudadano

## Historia de usuario
**Como** ciudadano en proceso de subsanación  
**Quiero** visualizar el paso de datos específicos precargado con mi información original  
**Para** ver el contexto y editar solo los campos marcados para subsanar

## Descripción funcional
El paso 2 (datos específicos) se carga precargado con todos los datos introducidos en la solicitud original. Este paso es idéntico en estructura y configuración al paso 2 de la solicitud original, presentando los mismos campos y validaciones.

El sistema presenta los campos mostrando:
- Campos marcados por el tramitador para subsanar: editables
- Campos no marcados: en modo solo lectura con sus valores originales

Esto permite al ciudadano ver el contexto completo de su solicitud mientras edita únicamente lo requerido.

Los demás pasos del formulario (paso 4 documentos, paso 5 declaraciones, paso 3 autorizaciones, etc.) se cargan según la configuración del CCP (Catálogo de Configuración de Procedimientos) específica para la acción telemática de subsanación. Estos pasos se presentan vacíos y deben ser completados por el ciudadano según los requisitos configurados. No se muestran elementos de la solicitud original en estos pasos.

El paso de identificación/interesados no es modificable en subsanación, siendo estos datos inmutables.

## Flujo principal
1. Sistema carga la subsanación
2. Sistema carga paso 2 con datos de solicitud original
3. Sistema marca visualmente campos a subsanar como editables
4. Sistema muestra campos no marcados en solo lectura
5. Ciudadano visualiza paso 2 con datos precargados
6. Sistema carga demás pasos según configuración CCP de subsanación
7. Ciudadano navega a otros pasos configurados
8. Sistema presenta pasos adicionales vacíos para completar

## Criterios de aceptación

### Escenario 1: Carga de paso 2 con datos originales

**Dado** que el ciudadano accede a subsanación  
**Cuando** el sistema carga los datos de la subsanación
**Entonces** el sistema carga el paso 2 con datos de la solicitud original  
**Y** todos los valores originales están presentes

### Escenario 2: Indicación visual de campos editables

**Dado** que el paso 2 se presenta al ciudadano  
**Cuando** visualiza los campos  
**Entonces** el sistema indica visualmente los campos marcados para subsanar  
**Y** diferencia claramente editables de solo lectura  
**Y** usa convenciones visuales consistentes

### Escenario 3: Campos en solo lectura mantienen valores

**Dado** que un campo no está marcado para subsanar  
**Cuando** el ciudadano visualiza el paso 2  
**Entonces** el sistema muestra el campo en modo solo lectura  
**Y** muestra el valor original  
**Y** no permite la edición

### Escenario 4: Carga de otros pasos según CCP

**Dado** que el ciudadano navega a otros pasos  
**Cuando** accede a paso de documentos o declaraciones  
**Entonces** el sistema carga el paso según configuración CCP de subsanación  
**Y** el paso se presenta vacío para completar  
**Y** no precarga datos de solicitud original

### Escenario 5: Navegación entre pasos

**Dado** que el ciudadano está en un paso  
**Cuando** navega a otro paso  
**Entonces** el sistema permite la navegación  
**Y** mantiene cambios realizados en paso 2  
**Y** presenta pasos configurados correctamente
